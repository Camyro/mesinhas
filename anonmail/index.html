TME
AnonMail TME Architecture

Transparent Multi-Mode Execution — uma página HTML estática age como intermediária. Ela lê parâmetros da URL, executa o envio via o provedor configurado, e retorna o resultado no hash da URL para o WebViewer do Kodular. Sem backend próprio, sem servidor dedicado.

Modos disponíveis
anon
Anônimo
EmailJS interno. Limite global de 180/mês. Sem credenciais.
emailjs
EmailJS
Suas próprias credenciais. Você controla o limite e o template.
formspree
Formspree
Envia via Formspree. Chega no e-mail da sua conta.

Resposta via hash da URL

A página grava o resultado em window.location.hash. No Kodular, leia via WebViewerTME.CurrentUrl no evento PageLoaded.

✓ #result=success&mode=anon
✓ #result=success&mode=emailjs
✗ #result=error&msg=Motivo+do+erro
Guia de integração — Kodular
1
Adicione um WebViewer invisível

No Designer, arraste um WebViewer para a tela. Defina Visible → false. Nomeie-o WebViewerTME.

💡 Se já usa um WebViewer para outra coisa, crie um segundo componente dedicado só ao TME.
2
Crie uma variável global de controle

Crie uma variável global booleana tme_aguardando com valor inicial false. Ela evita reagir a carregamentos não relacionados no evento PageLoaded.

3
Monte a URL e abra no WebViewer

Use Web → UriEncode em cada valor antes de concatenar. Marque tme_aguardando = true antes de chamar GoToUrl. A base é mesinhas.vercel.app/email/tme/.

⚠ Use Web.UriEncode em cada campo individualmente — nunca concatene texto puro com &, =, espaços ou acentos.
4
Leia o resultado no evento PageLoaded

Em WebViewerTME.PageLoaded, verifique tme_aguardando. Leia WebViewerTME.CurrentUrl e separe o hash com # (posição 2). Se contiver result=success mostre sucesso; caso contrário extraia o erro com segment(hash, "msg=", 2) e decodifique com Web.UriDecode. Marque tme_aguardando = false.

💡 segment(texto, separador, 2) é o bloco Text → segment do Kodular.
5
Desabilite o botão durante o envio

Desabilite o botão antes de chamar GoToUrl e reabilite dentro do PageLoaded, após tratar o resultado.

Guia de integração
1
Como funciona

Um fetch com method: 'POST' e Content-Type: application/json. A API retorna JSON com success: true/false. CORS habilitado para qualquer origem.

2
Copie a função sendViaAPI

Cole no seu projeto. Retorna uma Promise — use await ou .then/.catch.

const API_URL = "https://mesinhasserver.vercel.app/api/email";

async function sendViaAPI(params) {
  const res = await fetch(API_URL, {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify(params)
  });
  const data = await res.json();
  if (!data.success) throw new Error(data.error || 'Erro desconhecido');
  return data;
}
3
Chame no clique do botão
document.getElementById('btn-enviar').addEventListener('click', async () => {
  const btn = document.getElementById('btn-enviar');
  btn.disabled    = true;
  btn.textContent = 'Enviando...';
  try {
    await sendViaAPI({
      mode:    'anon',
      to:      document.getElementById('input-para').value,
      subject: document.getElementById('input-assunto').value,
      message: document.getElementById('input-mensagem').value
    });
    alert('✓ E-mail enviado!');
  } catch(e) {
    alert('✗ Erro: ' + e.message);
  } finally {
    btn.disabled    = false;
    btn.textContent = 'Enviar';
  }
});
✓ O finally garante que o botão seja sempre reabilitado, mesmo em erros.
4
Exemplo com mode emailjs

Para usar suas próprias credenciais passe os parâmetros extras. O mesmo padrão vale para mode: 'formspree' com fs_id.

await sendViaAPI({
  mode:         'emailjs',
  to:           'destino@email.com',
  subject:      'Olá!',
  message:      'Mensagem de teste via API.',
  from_name:    'João',
  from_email:   'joao@email.com',
  ejs_key:      'SUA_PUBLIC_KEY',
  ejs_service:  'service_xxxxxxx',
  ejs_template: 'template_xxxxxxx'
});
1
Como funciona

Use o componente Web com PostText passando o body JSON. A resposta chega no evento Web.GotText como string JSON — verifique se contém "success":true.

2
Configure o componente Web

Arraste um componente Web invisível e nomeie WebEmail. Defina o header Content-Type: application/json via Web.RequestHeaders. URL: https://mesinhasserver.vercel.app/api/email.

3
Monte o JSON e envie

Use Text → join para montar a string JSON com aspas duplas. Desabilite o botão antes de chamar WebEmail.PostText.

⚠ Exemplo anon: {"mode":"anon","to":"EMAIL","subject":"ASSUNTO","message":"MSG"} — substitua pelos valores dos seus TextBoxes.
4
Leia a resposta no evento GotText

Em WebEmail.GotText, cheque responseCode = 200. Para extrair o erro: segment(responseContent, "error\":\"", 2) → segment(resultado, "\"", 1). Reabilite o botão ao terminar.

💡 responseCode contém o HTTP status (200, 400, 429, 500) — mais simples que parsear o JSON.
