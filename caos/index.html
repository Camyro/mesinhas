TME
Caos AI — TME Architecture

Transparent Multi-Mode Execution — uma página HTML estática age como intermediária. Ela lê a mensagem da query string, chama a IA e retorna a resposta no hash da URL para o WebViewer do Kodular. Sem backend próprio.

Parâmetros da URL
ParâmetroReq.Descrição
message✓Mensagem do usuário para a IA
system—System prompt customizado (opcional)
model—Forçar engine: cerebras | huggingface | mistral

A URL base é mesinhas.vercel.app/caos/tme/. Use UriEncode em cada valor antes de concatenar.

Resposta via hash da URL

A página grava o resultado em window.location.hash. Leia via WebViewerTME.CurrentUrl no evento PageLoaded.

✓#result=success&reply=Resposta+da+IA&model=id
✗#result=error&msg=Motivo+do+erro
Guia de integração — Kodular
1
Adicione um WebViewer invisível

Arraste um WebViewer para a tela. Defina Visible → false. Nomeie WebViewerTME.

2
Variável de controle

Crie uma global booleana tme_aguardando iniciando em false para evitar falsos positivos no PageLoaded.

3
Monte a URL e abra

Use Web.UriEncode no valor de message antes de concatenar. Marque tme_aguardando = true e chame GoToUrl.

⚠ Nunca concatene texto com acentos, espaços ou & sem encodar — quebrará o parâmetro.
4
Leia o resultado em PageLoaded

Verifique tme_aguardando. Leia CurrentUrl, separe com # (pos. 2). Se contiver result=success extraia reply= e decodifique com Web.UriDecode. Marque tme_aguardando = false.

API
Caos AI — API

POST para https://mesinhasserver.vercel.app/api/chat — envia uma mensagem para a IA e recebe a resposta em JSON. Limite de 20 requisições / minuto por IP e 200 / dia global. CORS habilitado para qualquer origem.

Endpoint
MétodoURLContent-Type
POSTmesinhasserver.vercel.app/api/chatapplication/json
Body (JSON)
CampoReq.Descrição
message✓Mensagem do usuário para a IA
system—System prompt customizado (opcional)
forceEngine—Forçar engine: cerebras | huggingface | mistral
webSearch—Forçar web search via Mistral (boolean)

O campo message com valor __model_info__ retorna o nome do modelo e o uso diário sem chamar a IA — não conta no limite.

Respostas
200{ "reply": "Resposta da IA", "model": "...", "engine": "cerebras" }
400{ "reply": "Mensagem inválida", "model": "..." }
429{ "reply": "⚠️ Limite atingido. Aguarde 1 minuto.", "model": "..." }
500{ "reply": "❌ Erro interno da API", "model": "..." }
Exemplo de integração
1
Função base

Retorna uma Promise com a resposta da IA.

const API_URL = "https://mesinhasserver.vercel.app/api/chat";

async function askAI(message, system = "", forceEngine = null) {
  const res = await fetch(API_URL, {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify({ message, system, forceEngine })
  });
  const data = await res.json();
  if (res.status === 429) throw new Error('Limite atingido');
  if (!res.ok) throw new Error(data.reply || 'Erro desconhecido');
  return data; // { reply, model, engine }
}
2
Chamada no clique do botão
btn.addEventListener('click', async () => {
  btn.disabled = true;
  try {
    const { reply, model, engine } = await askAI(
      inputEl.value,
      "",
      "cerebras" // ou "huggingface" / "mistral" / null (auto)
    );
    outputEl.textContent = reply;
  } catch(e) {
    outputEl.textContent = 'Erro: ' + e.message;
  } finally {
    btn.disabled = false;
  }
});
✓ Passe forceEngine: null para deixar a API escolher automaticamente.
1
Configure o componente Web

Arraste um Web invisível e nomeie WebIA. Defina o header Content-Type: application/json. URL: https://mesinhasserver.vercel.app/api/chat.

2
Monte o JSON e envie

Use Text → join para montar a string JSON. Inclua o campo forceEngine com o valor desejado (cerebras, huggingface ou mistral). Desabilite o botão antes de chamar WebIA.PostText.

⚠ Exemplo: {"message":"OLÁ","forceEngine":"cerebras"}
3
Leia a resposta em GotText

Em WebIA.GotText, cheque responseCode = 200. Para extrair: segment(responseContent, "reply\":\"", 2) → segment(resultado, "\"", 1).

💡 Se responseCode = 429, mostre "aguarde 1 minuto" antes de reabilitar.
